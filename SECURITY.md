# Security policy

## Supported versions

The latest tagged major release is supported. Earlier releases receive fixes only when the issue also affects the latest release.

## Reporting a vulnerability

Please email `alfred-charles@agentmail.to` with:

- affected version or commit;
- minimal reproduction;
- expected and observed behavior;
- potential impact.

Do not include repository secrets, customer data, private workflow contents, credentials, or exploit data that could harm another party. We will acknowledge a report when the inbox is operationally available. This project does not offer a paid bug bounty.

## Scope

Workflow Guard is a focused static signal. It is not a certification and does not guarantee that a workflow or repository is secure.
